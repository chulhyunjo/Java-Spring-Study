# 02.JVM의 메모리 구조

> class area

- 클래스 원형 로딩
  - Field 정보
  - Method 정보
  - 타입 정보
  - 상수 풀



> method stack

- 메서드들의 실행 공간
  - thread 별로 별도 관리
  - 메서드 호출 순서대로 쌓이는 구조
  - 메서드 프레임에 로컬변수도 쌓이는 구조
  - 로컬 변수는 선언 된 영역을 벗어나면 삭제



> heap

- 객체를 저장하기 위한 영역
  - thread에 의해 공유
  - 생성된 객체는 프로그래머가 삭제할 수 없고 GC만 제어 가능
